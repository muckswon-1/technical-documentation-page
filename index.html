Introduction

Django is a high-level Python web framework that encourages rapid development and clean, pragmatic design. Built by experienced developers, it takes care of much of the hassle of web development, so you can focus on writing your app without needing to reinvent the wheel. It's free and open source.

Django is famously known to be Ridiculously fast, Reassuringly secure and Exceedingly scalable

Installation

First step to use Django is to install it. Python is a prerequisite to install Django. You can download the latest version of python from (python website). After python is installed you can proceed to install Django code.

The following code examples show how to install Django in a Linux/MacOs and Windows environment.

Linux/MacOs $ python -m pip install Django

Windows ...\> py -m pip install Django

Verifying Django is Installed

First run python by typing python in the terminal. Then import Django at the terminal by running the following code;

>>> import django >>> print(django.get_version())

Creating A project

You can create your first Django project by autogenerating some code; which is a collection of settings for an instance of Django, including Database configuration. In your directory of choice, run the following command to create a Django project:

Linux/MacOs $ django-admin startproject mysite

Windows ...\> django-admin startproject mysite

A Django project named 'mysite' contains the following:

  • The outer mysite/ root directory is a container for your project. Its name doesn't matter to Django; you can rename it to anything you like.
  • manage.py: A command-line utility that lets you interact with this Django project in various ways. You can read all the details about manage.py in django-admin and manage.py
  • The inner mysite/ directory is the actual Python package for your project. Its name is the Python package name you'll need to use to import anything inside it (e.g. mysite.urls).
  • mysite/__init__.py: An empty file that tells Python that this directory should be considered a Python package. If you're a Python beginner, read more about packages in the official Python docs.
  • mysite/settings.py: Settings/configuration for this Django project. Django settings will tell you all about how settings work.
  • mysite/urls.py: The URL declarations for this Django project; a “table of contents” of your Django-powered site. You can read more about URLs in URL dispatcher.
  • mysite/asgi.py: An entry-point for ASGI-compatible web servers to serve your project. See How to deploy with ASGI for more details.
  • mysite/wsgi.py: An entry-point for WSGI-compatible web servers to serve your project. See How to deploy with WSGI for more details.
The development Server

You need to verify that your Django works. Change to your project direct and run the following commands:

Linux/MacOs $ python manage.py runserver

Windows ...\> py manage.py runserver

You should see a “Congratulations!” page, with a rocket taking off when you visit http://127.0.0.1:8000/.

Database set up

If you open mysite/settings.py, the default database configuration is SQLite. For learning purposes SQLite is an easy choice. You can also use another database like PostgreSQL. To change to another database, you will need to install appropriate database bindings and change the keys according to your database. Once your database is set up, you need to create tables in the database before using them. Run the following command to do this;

Linux/MacOs $ python manage.py migrate

Windows ...\> py manage.py migrate
